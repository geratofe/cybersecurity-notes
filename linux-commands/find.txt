----------------------------------------
Useful Cybersecurity / Pentesting Uses of find
----------------------------------------

1. Find SUID files (Privilege Escalation)

SUID files run with the permissions of the file owner.
Attackers look for these to escalate privileges.

Command:
find / -perm -4000 -type f 2>/dev/null

Example output:
/usr/bin/passwd
/usr/bin/sudo

If a vulnerable SUID binary exists, it may allow root access.


2. Find world-writable files

World-writable files can be modified by anyone.

Command:
find / -type f -perm -o+w 2>/dev/null

Why important:
Attackers may replace or modify these files to gain access.


3. Find large files (may contain backups or logs)

Command:
find / -type f -size +100M 2>/dev/null

Why useful:
Large files sometimes contain backups, databases, or logs with credentials.


4. Find recently modified files

Command:
find / -type f -newermt "2024-01-01" 2>/dev/null

Why useful:
Helps identify recently changed files during an investigation.


5. Find hidden files

Hidden files start with a dot (.)

Command:
find / -type f -name ".*" 2>/dev/null

Why useful:
Attackers sometimes hide tools in hidden files.


6. Find configuration files

Command:
find / -type f -name "*.conf" 2>/dev/null

Why useful:
Configuration files may contain credentials, database connections, or API keys.


7. Find backup files

Command:
find / -type f -name "*backup*" 2>/dev/null

Why useful:
Backups sometimes contain sensitive data like passwords.


8. Find SSH keys

Command:
find / -type f -name "id_rsa" 2>/dev/null

Why useful:
Private SSH keys may allow login to other systems.
